# 🔧 Как работают два бота

**Подробное описание архитектуры и взаимодействия**

</div>

<br>

## 📐 Общая схема

```
┌──────────────────────────────────────────────────────────────┐
│                      Один процесс (bot.py)                   │
│                                                              │
│   ┌─────────────────────────┐   ┌──────────────────────────┐ │
│   │    🤖 Основной бот      │   │    🛡 Админ-бот          │ │
│   │                         │   │                          │ │
│   │  • Генерация фото       │──►│  • Уведомления о юзерах  │ │
│   │  • Генерация видео      │──►│  • Логи ошибок           │ │
│   │  • Вариации фото           │   │  • /stats, /users        │ │
│   └─────────────────────────┘   └──────────────────────────┘ │
│               ▲                             ▲                │
│               │                             │                │
│          Пользователи                  Администратор         │
└──────────────────────────────────────────────────────────────┘
```

> Оба бота запускаются **одной командой** `docker compose up` и работают параллельно в одном процессе через `asyncio.gather`.

## 🛡 Админ-бот

Бот для администратора. Работает в **двух режимах**: автоматическом и по команде.

---

### 🔔 Автоматические уведомления

Админ-бот **сам отправляет** сообщения администратору при событиях в основном боте.

## 🔗 Как боты связаны между собой

### Принцип работы

```python
# bot.py — точка входа

main_bot = Bot(token=MAIN_BOT_TOKEN)     # Основной бот
admin_bot = Bot(token=ADMIN_BOT_TOKEN)   # Админ-бот

# Основной бот получает доступ к объекту админ-бота
# чтобы отправлять через него уведомления
main_dispatcher = create_main_dispatcher(admin_bot)  # ← связь

# Оба запускаются параллельно
await asyncio.gather(
    main_dispatcher.start_polling(main_bot),
    admin_dispatcher.start_polling(admin_bot),
)
```

---

### Middleware-цепочка

Каждое сообщение от пользователя проходит через **3 слоя** обработки:

```
📩 Сообщение от пользователя
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│  1️⃣  DbSessionMiddleware                                │
│                                                         │
│  Открывает сессию к базе данных                         │
│  и передаёт её дальше по цепочке                        │
└─────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│  2️⃣  UserTrackingMiddleware                              │
│                                                         │
│  Проверяет: есть ли пользователь в базе?                │
│                                                         │
│  ❌ Нет → Сохраняет в БД                                │
│         → Отправляет "👤 Новый пользователь!"           │
│           через админ-бот                               │
│                                                         │
│  ✅ Да  → Пропускает дальше                             │
└─────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│  3️⃣  ErrorLoggingMiddleware                              │
│                                                         │
│  Оборачивает обработку в try/except                     │
│                                                         │
│  ❌ Ошибка → Отправляет "🚨 Ошибка в боте!"            │
│              через админ-бот                            │
│                                                         │
│  ✅ OK     → Пропускает дальше                          │
└─────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│  4️⃣  Handler                                            │
│                                                         │
│  Обработка команды: /start, генерация, вариации          │
└─────────────────────────────────────────────────────────┘
```

## 🗄 База данных

Оба бота используют **одну общую базу данных** (SQLite).

<table>
<tr>
<td width="50%">

### Таблица `users`

| Поле | Тип | Описание |
|:-----|:----|:---------|
| `id` | int | Первичный ключ |
| `telegram_id` | bigint | ID в Telegram |
| `username` | string | @username |
| `first_name` | string | Имя |
| `created_at` | datetime | Дата регистрации |

</td>
<td width="50%">

### Таблица `generations`

| Поле | Тип | Описание |
|:-----|:----|:---------|
| `id` | int | Первичный ключ |
| `user_id` | FK → users | Кто запросил |
| `gen_type` | string | `image` / `video` / `variation` |
| `model` | string | Какая модель |
| `prompt` | string | Текст промпта |
| `status` | string | `done` / `error` |
| `created_at` | datetime | Дата генерации |

</td>
</tr>
</table>
